# 수면 무호흡 분석 서비스 (Hypoxic Burden Calculator)

## 개요

NeuroHub의 **수면 정밀 분석 서비스**는 수면다원검사(PSG) 데이터로부터 **저산소 부담(Hypoxic Burden, HB)** 및 주요 수면 호흡 지표를 자동 계산하는 AI 분석 파이프라인입니다.

- **알고리즘**: Sutherland et al., *Sleep* (2022) 기반 HB 정량 분석
- **구현**: 순수 JavaScript (Node.js) — Python 의존 없음
- **컨테이너**: `neurohub/sleep-apnea:1.0.0` (Docker)
- **모달리티**: PSG (수면다원검사)

## 입력 파일 형식

### 1. SpO₂ 산소포화도 CSV (필수)

```csv
relative position (hh:mm:ss.ms),absolute position (hh:mm:ss.ms),OSat ("%")
00:00:00.000,22:02:23.000,98
00:00:01.000,22:02:24.000,98
00:00:02.000,22:02:25.000,97
...
```

- **파일명 패턴**: `osat`, `spo2`, `o2sa`, `alltrends.*osat`, `oxygen` (대소문자 무관)
- **형식**: CSV (쉼표/탭/세미콜론 자동 감지)
- **컬럼**: 시간(HH:MM:SS 또는 초), SpO₂ 값 (0-100%)
- **자정 전환(Day Rollover)**: 자동 보정

### 2. 이벤트 리스트 TXT/CSV (필수)

```
Time        Epoch   Stage   Channel     Duration    Title
22:26:29    49      N2      SpO2        1:02.1      Oxygen Desaturation (-3.0%)
00:40:55    318     N1      Airflow     0:15.9      Obstructive Apnea (Lowest SpO2: 94.0%)
01:01:16    358     R       PTAF        0:24.3      Central Apnea (Lowest SpO2: 95.0%)
...
```

- **파일명 패턴**: `eventlist`, `event_list`, `events`
- **형식**: Natus PSG 표준 (UTF-16 지원), CSV, 탭 구분
- **이벤트 유형**: Apnea, Hypopnea, Desaturation, Arousal (자동 필터링)
- **기간 형식**: `MM:SS.ms` 또는 초 단위

### 3. 수면단계 Hypnogram CSV (선택)

```csv
position (epoch),absolute position (hh:mm:ss.ms),Default Staging Set ("stage")
1,22:02:23.000,?
49,22:26:29.000,N2
100,22:51:30.000,N3
...
```

- **파일명 패턴**: `hypnogram`, `hypn`, `alltrends.*hypn`, `sleep.*stage`
- **수면단계 코드**: W, N1, N2, N3, N4, R, ? (자동 변환)
- 미제공 시 수면시간은 전체 기록시간에서 추정

## 출력 지표 (Features)

| 지표 | 설명 | 단위 | 임상 의미 |
|------|------|------|-----------|
| **HB** | 저산소 부담 (Hypoxic Burden) | %·min/hr | 주요 예후 지표 |
| **AHI** | 무호흡-저호흡 지수 | events/hr | 중증도 분류 기준 |
| **OAI** | 폐쇄성 무호흡 지수 | events/hr | |
| **HI** | 저호흡 지수 | events/hr | |
| **ODI** | 산소 탈포화 지수 | events/hr | |
| **REI** | 호흡이벤트 지수 | events/hr | |
| **ArI** | 각성 지수 | events/hr | |
| **sleep_hours** | 총 수면시간 | 시간 | |
| **sleep_efficiency** | 수면효율 | % | (수면시간/기록시간)×100 |
| **severity** | 중증도 분류 | 텍스트 | 정상/경증/중등도/중증 |

### 중증도 분류 기준

| AHI 범위 | 분류 | 한국어 |
|----------|------|--------|
| < 5 | Normal | 정상 |
| 5 – 14.9 | Mild | 경증 |
| 15 – 29.9 | Moderate | 중등도 |
| ≥ 30 | Severe | 중증 |

## 시각화 데이터 (Maps)

컨테이너는 시각화를 위한 원시 데이터를 `maps` 필드로 함께 반환합니다:

| Map | 내용 | 용도 |
|-----|------|------|
| `spo2_timeseries` | 전체 SpO₂ 시계열 (`t`, `values`) | SpO₂ 추이 그래프 |
| `avg_response` | 이벤트 앙상블 평균 SpO₂ 응답 | HB 시각화 |
| `events` | 호흡이벤트 목록 (유형/시작/기간) | 이벤트 타임라인 |
| `sleep_stages` | 수면단계 어노테이션 (`t`, `annotations`) | 수면단계 그래프 |

## 알고리즘 상세

### HB (Hypoxic Burden) 계산

```
HB = Σ(이벤트별 탈포화 면적) / 총_수면시간   [%·min/hr]
```

**계산 과정:**
1. 각 무호흡/저호흡 이벤트 종료 시점을 기준으로 분석 창(window) 설정
   - 창 시작: 이벤트 종료 **-5초**
   - 창 종료: 이벤트 종료 **+45초**
2. 이벤트 시작 전 100초 구간에서 기준선(baseline) SpO₂ 산출 (최대값)
3. 기준선 아래로 떨어진 SpO₂ 면적(area-under-curve)을 적분
4. 전체 이벤트의 면적 합산 후 총 수면시간으로 정규화

### 앙상블 평균 (Ensemble Averaging)
- 모든 이벤트를 종료 시점 기준으로 정렬
- ±120초 창에서 시점별 평균 ± 표준편차 계산
- 12점 이동평균 필터 적용

### 컨테이너 실행 흐름

```
1. 사용자가 웹에서 PSG 파일 업로드
2. FastAPI → Request 생성 (상태: CREATED)
3. POST /requests/{id}/execute → Run/TechniqueRun 생성
4. Celery 워커 → LocalContainerRunner 호출
5. Docker 컨테이너 실행:
   - /input 에서 SpO₂, EventList, Hypnogram 파일 탐색
   - hb-calculator.js 실행
   - /output/results.json 저장
   - NEUROHUB_OUTPUT JSON → stdout
6. 결과 파싱 → DB 저장 → 상태: COMPUTING → QC → FINAL
7. 프론트엔드에서 결과 표시
```

## 독립 실행 (개발/테스트)

### hb-service API 서버

```bash
cd hb-service
npm install
npm start        # http://localhost:3000
```

**API 엔드포인트:**

```
POST /api/hb
  Content-Type: multipart/form-data
  필드:
    spo2       (필수) - SpO₂ CSV 파일
    eventlist  (필수) - 이벤트 리스트 TXT/CSV
    hypn       (선택) - 수면단계 CSV
```

**응답**: 전체 HB 계산 결과 JSON (지표, 시계열, 이벤트 목록 포함)

**웹 인터페이스**: `http://localhost:3000` 에서 파일 업로드 UI 사용 가능

### Docker 컨테이너 (로컬 테스트)

```bash
# 빌드
cd containers/sleep-apnea
docker build -t neurohub/sleep-apnea:1.0.0 .

# 실행 (샘플 데이터)
docker run --rm \
  -v /path/to/input:/input \
  -v /path/to/output:/output \
  neurohub/sleep-apnea:1.0.0
```

`/input` 디렉토리에 SpO₂ CSV와 이벤트 리스트 파일 배치 후 실행.

## 사용자 워크플로우 (프론트엔드)

### 1단계: 서비스 선택
사용자 대시보드 → 서비스 카탈로그 → **수면 정밀 분석 서비스** 선택

### 2단계: 환자 정보 입력
- 환자 참조번호 (필수)
- 나이, 성별, BMI (필수)
- 수면 설문 점수 (필수)
- 동반질환, 복용약물 (선택)

### 3단계: 파일 업로드
지원 확장자: `.csv`, `.txt`, `.edf`, `.zip`

| 파일 | 내용 | 필수 여부 |
|------|------|-----------|
| SpO₂ CSV | 산소포화도 시계열 | 필수 |
| EventList TXT | 호흡이벤트 목록 | 필수 |
| Hypnogram CSV | 수면단계 어노테이션 | 선택 |

### 4단계: 분석 실행
업로드 완료 후 **분석 시작** 클릭 → 자동 파이프라인 실행

### 5단계: 결과 확인
요청 상세 페이지에서 분석 결과 확인:
- HB, AHI 등 핵심 지표
- 중증도 분류 (정상/경증/중등도/중증)
- QC 점수
